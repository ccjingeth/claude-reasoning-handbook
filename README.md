# Claude 深度推理强制执行手册

[![Version](https://img.shields.io/badge/version-v2.2-blue.svg)](https://github.com/ccjingeth/claude-reasoning-handbook/blob/main/CLAUDE.md)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

这是一份帮助 AI 模型进行深度推理的系统级手册，确保每次回答都经过严格的逻辑验证。

## 快速开始

将此手册放置在以下位置之一：

- **全局规则**（推荐）：`~/.claude/rules/CLAUDE.md`
- **项目级**：`/path/to/project/CLAUDE.md`

## 核心特性

### 执行流程图
- 明确的规则执行顺序
- 快速通道机制（简单任务跳过完整流程）
- 违规征兆自查表

### 三次停顿原则

每个回答前，强制停顿三次：

1. **读题停顿** — 区分字面意思和真实意图
2. **拆解停顿** — 分成 3-5 块，可独立验证
3. **验证停顿** — 核心数字、逻辑、假设必须重新推导

## 主要规则

- **规则 1**：读题 — 区分字面意思和真实意图
- **规则 1.5**：模糊需求必须澄清
- **规则 2**：拆解 — 分成可独立验证的小块
- **规则 2.5**：识别和明示权衡
- **规则 3**：识别风险 — 优先验证最容易错的地方
- **规则 4**：验证 — 区分需要重算的数字
- **规则 7**：常见陷阱（规则4的补充）
- **规则 5**：区分事实和猜测
- **规则 6**：输出顺序 — 先答案，后推理，最后风险
- **规则 8**：自测清单（6个问题）

## 渐进式掌握路径

### 第一级：核心三规则（必须掌握）
- 规则1（读题）+ 规则4（验证）+ 规则6（输出）
- 目标：比90%的回答更严谨

### 第二级：扩展四规则（提升质量）
- 规则1.5、2、3、5
- 目标：可以处理复杂任务

### 第三级：精通全部（专家级）
- 规则2.5、7、8
- 目标："零失误"级别

## 测试用例

手册包含 10 个测试用例，覆盖所有关键规则：
- 数字陷阱（百分比计算）
- 猜测 vs 事实
- 拆解验证
- 模糊需求澄清
- 权衡识别
- 输出结构
- 微妙错误（重复表述）
- 读题（隐含约束）
- 自测清单

**为什么需要这份手册？**

AI 模型在推理时容易出现以下问题：
- 看到数字不验证，直接相信
- 遇到模糊需求不澄清，直接猜测
- 给出答案但没有说明风险
- 推理过程在前，结论在后（用户要翻到最后才知道答案）

这份手册通过**强制规则**解决这些问题，让每次回答都经过严格验证。
